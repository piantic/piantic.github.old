WinAPI 및 유닉스 시스템을 이용한 퍼즐 놓기 게임

요약

이 문서는 UNIX System 과목의 Term 최종 보고서로써, 3월부터 6월 4일까지 진행된 과제의 개발목적, 게임기능, 소스분석, 완성된 화면까지의 모든 것을 담고있다.

수행범위 : TCP/IP를 이용한 Server(unix) -Client모델 기반의 GUI 프로그램 개발

수행기간 : 2008.03.20 ~ 2008.06.04

개발 목적

이번 학기의 Unix System과 지난 학기의 C programming 배운 지식을 바탕으로 이를 응용하여 멀티유저지원이 있는 채팅과 게임이 연동되는 프로그램을 만드는 것을 목표로 하였다. 채팅과 게임을 구현하기 위해서는 TCP/IP 이용한다. Windows API를 사용 기본 interface 및 기타 정보를 표현하였다. 강의내용을 바탕으로 독립적이고 독창적인 코딩으로 학기동안 학습한 내용의 복습 및 이해도를 높이는데 중점을 두어 개발하였다. 조별 프로젝트를 수행함으로써 모듈화 프로그래밍에 대한 경험을 해보며, 이를 통해 상호간의 의사전달능력 및 협동심을 높인다.

게임과 채팅의 세부 내용

게임 관련

1. 9x9의 판에 블록들을 두 사람이 번갈아 가며 놓는데, 겹치며 놓을 수는 없고, 더 이상 놓을 자리가 없으면 진다. 결국 다른 사람이 놓을 자리가 없게 만들면 이긴다.

2. 키조작 - 회전, B : undo, Space Bar : 바꾸기, Enter : 블록입력

3. 블록은 테트리스 블록과 동일하다.

4. 멈춤 기능을 추가해서 사용자가 급한 용무가 있을 때 사용할 수 있도록 한다.

채팅

1. 로그인할　때 닉네임을 입력하도록 하며, 로그인하지 않으면 게임이 불가능하도록 한다.

2. 게임 중에도 대화가 가능하도록 하고, 게임 도중 닉네임을 임의로 바꿀 수 있도록 하며 이는 게임에 영향을 미치지 않도록 한다.
